## 1. Schema 命名規則

FastAPI 官方採用「**後綴區分用途**」的命名方式：

| 後綴 | 用途 |
|------|------|
| `Base` | 共用欄位的基底類別 |
| `Create` | 建立資源時的輸入 |
| `Update` | 更新資源時的輸入 |
| `Read` / `Response` | API 回應格式 |
| `InDB` | 資料庫層的 model |

```python
class UserBase(BaseModel):
    email: str
    name: str

class UserCreate(UserBase):
    password: str

class UserUpdate(BaseModel):
    name: str | None = None

class UserRead(UserBase):
    id: int
    created_at: datetime

class UserInDB(UserRead):
    hashed_password: str
```

> ⚠️ 避免直接用 `User` 命名 schema，容易和 SQLAlchemy ORM model 衝突

## 2. JWT Token

### Login 回傳格式

Token 是完整字串，**不需要拆解**，直接放 `str`：

```python
class TokenResponse(BaseModel):
    access_token: str
    token_type: str = "bearer"
    expires_in: int | None = None

# 有 refresh token 時
class TokenPairResponse(BaseModel):
    access_token: str
    refresh_token: str
    token_type: str = "bearer"
    expires_in: int | None = None
```

### Access Token vs Refresh Token

| | Access Token | Refresh Token |
|---|---|---|
| 有效期 | 短（分鐘～小時）| 長（天～週）|
| 用途 | 存取 API | 只用來換新 access token |
| 儲存位置 | 記憶體 | HttpOnly Cookie |
| 是否每次帶 | ✅ 每次 request | ❌ 只在換 token 時 |

### 為什麼 Refresh Token 還算安全？

- Refresh token 存在 **HttpOnly Cookie**，JS 讀不到，防 XSS
- **Token Rotation**：每次換 token 時舊的作廢，偵測到重用就強制登出
- 存在資料庫，可以**主動撤銷**
- 安全靠多層機制疊加：短效 token + HttpOnly + Rotation + HTTPS

### HS256 vs RS256

| | HS256 | RS256 |
|---|---|---|
| 類型 | 對稱式 | 非對稱式 |
| 簽名 | 同一把 secret key | 私鑰 |
| 驗證 | 同一把 secret key | 公鑰 |
| 適合 | 單體應用、小專案 | 微服務、多服務驗證 |

```python
# HS256
token = jwt.encode(payload, SECRET_KEY, algorithm="HS256")

# RS256（用不同 key）
token = jwt.encode(payload, PRIVATE_KEY, algorithm="RS256")   # 簽發
payload = jwt.decode(token, PUBLIC_KEY, algorithms=["RS256"])  # 驗證
```

## 3. SQLAlchemy Async

### 基本設定

```python
# database.py
from sqlalchemy.ext.asyncio import AsyncSession, create_async_engine
from sqlalchemy.orm import DeclarativeBase, sessionmaker

engine = create_async_engine("postgresql+asyncpg://user:pass@localhost/db")

AsyncSessionLocal = sessionmaker(
    engine, class_=AsyncSession, expire_on_commit=False
)

class Base(DeclarativeBase):
    pass

async def get_db():
    async with AsyncSessionLocal() as session:
        yield session
```

### User Model 範例

```python
class User(Base):
    __tablename__ = "users"

    id: Mapped[int] = mapped_column(primary_key=True)
    email: Mapped[str] = mapped_column(String, unique=True)
    name: Mapped[str] = mapped_column(String)
    password_hash: Mapped[str] = mapped_column(String(128))

    def set_password(self, password: str) -> None:
        self.password_hash = hashpw(
            password.encode("utf-8"), gensalt()
        ).decode("utf-8")
```

### 常用 Query

```python
from sqlalchemy import select, update, delete

# 查單筆 by id（有快取）
user = await session.get(User, user_id)

# 查單筆 by 條件
result = await session.execute(
    select(User).where(User.email == "test@example.com")
)
user = result.scalar_one_or_none()

# 查多筆
result = await session.execute(
    select(User).where(User.is_active == True).order_by(User.id.desc())
)
users = result.scalars().all()

# 新增
user = User(email="new@example.com", name="Alice")
session.add(user)
await session.commit()
await session.refresh(user)  # 同步 DB 產生的 id 等欄位

# 更新（先查後改）
user = await session.get(User, user_id)
user.name = "Bob"
await session.commit()

# 刪除
await session.delete(user)
await session.commit()
```

### scalar 系列方法

| 方法 | 找不到 | 多筆 |
|---|---|---|
| `scalar_one()` | 報錯 | 報錯 |
| `scalar_one_or_none()` | `None` | 報錯 |
| `scalars().all()` | `[]` | 全部回傳 |
| `scalars().first()` | `None` | 只取第一筆 |

### commit vs flush

| | `flush` | `commit` |
|---|---|---|
| 寫入 DB | ✅ | ✅ |
| 結束 transaction | ❌ | ✅ |
| 可以 rollback | ✅ | ❌ |

> `flush` 常用於同一 transaction 內需要先拿到 id 再建關聯資料

### session.execute 為什麼先回傳 Result？

因為 `execute()` 是通用方法，不知道你想要什麼格式，所以先回傳 `Result` 讓你自己決定：

```python
result = await session.execute(select(User))
result.scalars().all()       # [User, User, User]
result.scalar_one_or_none()  # User | None
result.all()                 # [(User,), (User,)]  原始 tuple 格式
result.mappings().all()      # [{"id": 1, ...}]    dict 格式
```

## 4. Repository 模式

Session 從外面傳入，Repository 只負責 DB 操作：

```python
# repositories/user.py
class UserRepository:
    def __init__(self, session: AsyncSession):
        self.session = session

    async def get_by_id(self, user_id: int) -> User | None:
        return await self.session.get(User, user_id)

    async def get_by_email(self, email: str) -> User | None:
        result = await self.session.execute(
            select(User).where(User.email == email)
        )
        return result.scalar_one_or_none()

    async def create(self, email: str, name: str, password_hash: str) -> User:
        user = User(email=email, name=name, password_hash=password_hash)
        self.session.add(user)
        await self.session.commit()
        await self.session.refresh(user)
        return user

    async def update_user(self, user_id: int, data: UserUpdate) -> User | None:
        user = await self.session.get(User, user_id)
        if not user:
            return None
        # exclude_unset=True 只更新有傳入的欄位
        for field, value in data.model_dump(exclude_unset=True).items():
            setattr(user, field, value)
        await self.session.commit()
        await self.session.refresh(user)
        return user
```

> 職責分工：Router/Service 負責 hashing、業務邏輯；Repository 只負責 DB 操作

## 5. FastAPI Endpoint

### 基本結構

```python
from fastapi import APIRouter, Depends, HTTPException
from sqlalchemy.ext.asyncio import AsyncSession

router = APIRouter(prefix="/users", tags=["users"])
# prefix → 所有 endpoint 自動加上路徑前綴
# tags   → Swagger UI 分組，純文件用途
```

### CRUD Endpoints

```python
@router.get("/", response_model=list[UserRead])
async def get_users(db: AsyncSession = Depends(get_db)):
    return await UserRepository(db).get_all()

@router.get("/{user_id}", response_model=UserRead)
async def get_user(user_id: int, db: AsyncSession = Depends(get_db)):
    user = await UserRepository(db).get_by_id(user_id)
    if not user:
        raise HTTPException(status_code=404, detail="User not found")
    return user

@router.post("/", status_code=201, response_model=UserRead)
async def create_user(body: UserCreate, db: AsyncSession = Depends(get_db)):
    return await UserRepository(db).create(body)

@router.patch("/{user_id}", response_model=UserRead)
async def update_user(user_id: int, body: UserUpdate, db: AsyncSession = Depends(get_db)):
    user = await UserRepository(db).update_user(user_id, body)
    if not user:
        raise HTTPException(status_code=404)
    return user

@router.delete("/{user_id}", status_code=204)
async def delete_user(user_id: int, db: AsyncSession = Depends(get_db)):
    ...
```

### 參數類型

```python
@router.get("/{user_id}")          # user_id → path parameter
async def get_user(
    user_id: int,                  # path parameter（名稱需對應路徑）
    page: int = 1,                 # query parameter（?page=1）
    body: UserCreate,              # request body（Pydantic model）
    db: AsyncSession = Depends(get_db)  # dependency injection
):
```

> FastAPI 靠**型別**判斷參數角色，變數名稱（`body`/`req`/`data`）可以自由命名

### main.py 註冊 router

```python
from fastapi import FastAPI
from routers import user, auth

app = FastAPI()
app.include_router(user.router)
app.include_router(auth.router)
```

### 啟動

```bash
uvicorn app.main:app
#        │    │   └── main.py 裡的 app 變數（FastAPI 實例）
#        │    └────── app 資料夾裡的 main.py
#        └─────────── app 資料夾
```

啟動後自動產生：
- `http://localhost:8000/docs` → Swagger UI（可互動測試）
- `http://localhost:8000/redoc` → ReDoc 文件
- `http://localhost:8000/openapi.json` → 原始 OpenAPI schema

## 6. Depends 依賴注入

```python
# Depends 收到的是函式本身，不是執行結果
db: AsyncSession = Depends(get_db)   # ✅ 傳函式
db: AsyncSession = Depends(get_db()) # ❌ 立刻執行，拿到結果
```

每次 request 進來 FastAPI 才呼叫 `get_db()`，所以每個 request 都有獨立的 session。

Depends 可以嵌套：

```python
async def get_current_user(
    token: str = Depends(oauth2_scheme),
    db: AsyncSession = Depends(get_db)
) -> User: ...

@router.get("/me")
async def get_me(current_user: User = Depends(get_current_user)):
    ...
```

## 7. JWT 認證實作

### 為什麼用 HTTPBearer 而不是 OAuth2PasswordBearer？

- `OAuth2PasswordBearer` 預期登入用 **form data**（`username` + `password`）
- 一般 API 習慣用 **JSON body**（`email` + `password`）
- 兩者衝突，改用 `HTTPBearer` 只專注取出 token

### HTTPBearer 寫法

```python
from fastapi.security import HTTPBearer, HTTPAuthorizationCredentials

bearer_scheme = HTTPBearer()

async def get_current_user(
    credentials: HTTPAuthorizationCredentials = Depends(bearer_scheme),
    session: AsyncSession = Depends(get_db)
) -> User:
    token = credentials.credentials  # JWT 字串
    # credentials.scheme      → "Bearer"
    # credentials.credentials → "eyJhbGciOiJIUzI1NiJ9..."
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=["HS256"])
        user_id = payload.get("sub")
    except JWTError:
        raise HTTPException(status_code=401, detail="Invalid token")

    user = await UserRepository(session).get_by_id(user_id)
    if not user:
        raise HTTPException(status_code=401)
    return user
```

### 流程

```
Client request
  Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...
        ↓
  bearer_scheme 解析 header
        ↓
  credentials.credentials = "eyJhbGciOiJIUzI1NiJ9..."
        ↓
  jwt.decode() 驗證
        ↓
  查 DB 取得 user
        ↓
  注入到 endpoint
```

### 保護 Endpoint

```python
# 需要驗證 → 加 Depends(get_current_user)
@router.get("/me")
async def get_me(current_user: User = Depends(get_current_user)):
    return current_user

# 不需要驗證 → 不加
@router.post("/auth/login")
async def login(body: LoginRequest): ...
```
