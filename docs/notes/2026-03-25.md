## 1. 🐳 Docker Compose — 建立 Redis Container

```yaml
services:
  redis:
    image: redis:7.2-alpine        # alpine = 輕量版
    container_name: my-redis
    restart: unless-stopped        # 崩潰自動重啟，除非手動停止
    ports:
      - "6379:6379"                # 主機port:容器port
    volumes:
      - redis_data:/data           # Named volume，資料持久化
      - ./redis.conf:/usr/local/etc/redis/redis.conf  # 掛載設定檔
    command: redis-server /usr/local/etc/redis/redis.conf
    networks:
      - app-network
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 30s
      timeout: 10s
      retries: 3

volumes:
  redis_data:
    driver: local

networks:
  app-network:
    driver: bridge
```

### 重要參數速查

| 參數 | 用途 |
|---|---|
| `image` | 指定 Docker image，建議鎖定版本 |
| `restart: unless-stopped` | 自動重啟，手動停止才不重啟 |
| `ports` | `主機:容器` port 對應 |
| `volumes` | 資料持久化，container 刪掉資料還在 |
| `healthcheck` | 定期檢查 container 是否健康 |
| `networks` | 讓多個 container 互相溝通 |

### 常用指令

```bash
docker compose up -d          # 背景啟動
docker compose ps             # 查看狀態
docker exec -it my-redis redis-cli  # 進入 Redis CLI
docker compose down           # 停止並移除
docker compose down -v        # 同時刪除 volume（資料消失）
```

## 2. 🗄️ Redis 基本概念

Redis 是一個 **key-value store**，用 key 查 value：

```bash
SET user:1 "Alice"      # 存值
GET user:1              # 取值 → "Alice"
DEL user:1              # 刪除
EXPIRE user:1 3600      # 設定過期時間（秒）
TTL user:1              # 查看剩餘時間
KEYS refresh_token:*    # 查所有符合的 key
```

> ⚠️ Redis **只能用 key 查 value**，不能反過來用 value 查 key

## 3. 💾 Cache-Aside Pattern（Lazy Loading）

最常見的快取策略，**應用程式自己管理快取**。

### Read 流程
```
App → 查 Redis → Cache Hit  → 直接回傳
              → Cache Miss → 查 DB → 存進 Redis → 回傳
```

### Write 流程
```
App → 寫入 DB → 刪除 Cache（讓下次 read 重新載入）
```

### 程式碼範例

```python
async def get_user(user_id: str, redis, db):
    cache_key = f"user:{user_id}"

    # 1. 先查 Cache
    cached = await redis.get(cache_key)
    if cached:
        return json.loads(cached)  # Cache Hit ✅

    # 2. Cache Miss → 查 DB
    user = await db.query(user_id)

    # 3. 寫入 Cache，設 TTL
    await redis.set(cache_key, json.dumps(user), ex=3600)
    return user
```

### 快取策略比較

| 策略 | 誰管快取 | 適合場景 |
|---|---|---|
| **Cache-Aside** | 應用程式 | 讀多寫少，入門首選 |
| **Read-Through** | 快取層 | 簡化應用邏輯 |
| **Write-Through** | 快取層 | 資料一致性要求高 |
| **Write-Behind** | 快取層 | 寫入頻繁、可接受延遲 |

> 💡 **初學者學習順序**：Cache-Aside → Write-Through → TTL 調校 → Write-Behind → Cache Stampede 防護

## 4. 🔌 Redis 連線設定（FastAPI）

### Connection Pool（正確做法）

```python
# core/redis.py
from redis.asyncio import Redis, ConnectionPool
from typing import AsyncGenerator

pool: ConnectionPool = None

async def init_redis_pool():
    global pool
    pool = ConnectionPool.from_url(
        "redis://localhost:6379",  # 本機開發
        # "redis://my-redis:6379"  # 同一 docker network
        max_connections=10,
        decode_responses=True
    )

async def close_redis_pool():
    await pool.disconnect()

# ✅ 正確：用 async with 確保連線用完歸還
async def get_redis() -> AsyncGenerator[Redis, None]:
    async with Redis(connection_pool=pool) as redis:
        yield redis
```

### Lifespan 管理生命週期

```python
# main.py
from contextlib import asynccontextmanager
from fastapi import FastAPI

@asynccontextmanager
async def lifespan(app: FastAPI):
    await init_redis_pool()  # 啟動時：初始化
    yield                    # ← FastAPI 在這裡運行
    await close_redis_pool() # 關閉時：釋放資源

app = FastAPI(lifespan=lifespan)
```

### 與 SQLAlchemy 對比

| | SQLAlchemy | Redis |
|---|---|---|
| Pool 建立 | `create_async_engine()` | `ConnectionPool.from_url()` |
| 生命週期 | `lifespan` | `lifespan` |
| 注入方式 | `Depends(get_db)` | `Depends(get_redis)` |

## 5. 🔑 JWT + Refresh Token 機制

### 兩個 Token 的職責

```
Access Token  — 短效（15分鐘），每次 API 請求帶著走
Refresh Token — 長效（7天），用來換發新的 Access Token
```

### 為什麼要分兩個？

```
Access Token 活 7天  → 被偷走就被攻擊 7 天，無法撤銷 ❌
Access Token 活 15分 → 被偷走最多 15 分鐘傷害
Refresh Token 存 Redis → 隨時可以 DEL 強制登出，立刻失效 ✅
```

### JWT Payload

```python
payload = {
    "iss": settings.JWT_ISS,        # 誰發的
    "sub": str(user_id),            # 用戶 ID（取出用 payload["sub"]）
    "exp": datetime.now(UTC) + timedelta(minutes=15),  # 過期時間
}
```

### Token 工具函數

```python
def create_access_token(user_id: uuid.UUID) -> str:
    payload = {
        "sub": str(user_id),
        "exp": datetime.now(UTC) + timedelta(minutes=15),
    }
    return jwt.encode(payload, settings.JWT_SECRET, algorithm=settings.JWT_ALGORITHM)

def verify_token(token: str) -> dict:
    try:
        payload = jwt.decode(token, settings.JWT_SECRET, [settings.JWT_ALGORITHM])
        return payload  # payload["sub"] 取出 user_id
    except JWTError:
        raise HTTPException(status_code=401, detail="Invalid token")
```

## 6. 🔄 Refresh Token 完整實作

### Redis 存什麼？

```
key:   "refresh_token:{token字串}"
value: "user_id"
TTL:   7天（604800秒）
```

> 為什麼 token 當 key、user_id 當 value？
> **你手上有什麼就用什麼當 key** — 前端帶來的是 token，所以 token 當 key 才查得到。

### Login

```python
@router.post("/login")
async def login(body: LoginRequest, redis: Redis = Depends(get_redis)):
    user_id = "user_123"  # 驗證帳密後取得

    access_token = create_access_token(user_id)
    refresh_token = create_refresh_token(user_id)

    # 存進 Redis
    await redis.set(
        f"refresh_token:{refresh_token}",
        user_id,
        ex=60 * 60 * 24 * 7
    )

    return {"access_token": access_token, "refresh_token": refresh_token}
```

### Refresh（含 Token Rotation）

```python
@router.post("/refresh")
async def refresh(body: RefreshRequest, redis: Redis = Depends(get_redis)):
    old_token = body.refresh_token
    redis_key = f"refresh_token:{old_token}"

    # 1. 驗證 JWT 格式
    payload = verify_token(old_token)
    user_id = payload["sub"]

    # 2. 確認 Redis 還有這個 token
    stored = await redis.get(redis_key)
    if not stored:
        raise HTTPException(status_code=401, detail="Token 已失效")

    # 3. Token Rotation：刪舊的，存新的
    await redis.delete(redis_key)

    new_access_token = create_access_token(user_id)
    new_refresh_token = create_refresh_token(user_id)

    await redis.set(
        f"refresh_token:{new_refresh_token}",
        user_id,
        ex=60 * 60 * 24 * 7
    )

    return {"access_token": new_access_token, "refresh_token": new_refresh_token}
```

### Logout

```python
@router.post("/logout")
async def logout(body: LogoutRequest, redis: Redis = Depends(get_redis)):
    await redis.delete(f"refresh_token:{body.refresh_token}")
    return {"message": "登出成功"}
```

## 7. 🔄 Token Rotation 是什麼？

每次使用 refresh token 就換一個新的，舊的刪掉：

```
登入       → refresh_token_A
用 A 換    → 刪掉 A，產生 refresh_token_B
用 B 換    → 刪掉 B，產生 refresh_token_C
...
A → B → C → D → ...（不斷輪替，像車輪滾動）
```

### 為什麼要 Rotation？

```
沒有 Rotation：token 被偷 → 攻擊者用到 7 天過期
有  Rotation：token 被偷 → 正常用戶換 token 時舊的已不在 Redis
                         → 攻擊者手上的 token 同時失效 ✅
```

## 8. 🛡️ Refresh Token 傳遞方式

| 方式 | 安全性 | 適合場景 |
|---|---|---|
| **Request Body** | ❌ XSS 風險 | 練習、Side Project |
| **HttpOnly Cookie** | ✅ JS 碰不到 | Web 前端正式產品 |
| **Header** | ❌ XSS 風險 | Mobile App |

### HttpOnly Cookie 寫法

```python
# 設定 Cookie
response.set_cookie(
    key="refresh_token",
    value=new_refresh_token,
    httponly=True,   # JS 無法存取
    secure=True,     # 只走 HTTPS
    samesite="lax",  # CSRF 保護
    max_age=60 * 60 * 24 * 7
)

# 讀取 Cookie
@router.post("/refresh")
async def refresh(refresh_token: str = Cookie(None)):
    ...
```

## 9. ⚡ FastAPI 依賴注入與參數

### Depends — 保護 API

```python
# 驗證 Access Token
async def get_current_user(
    credentials: HTTPAuthorizationCredentials = Depends(HTTPBearer())
):
    token = credentials.credentials
    payload = verify_token(token)
    return payload["sub"]

# 加在單個 endpoint
@router.get("/profile")
async def get_profile(user_id: str = Depends(get_current_user)):
    ...

# 加在整個 router
router = APIRouter(
    prefix="/user",
    dependencies=[Depends(get_current_user)]  # 整個 router 都套用
)
```

### Endpoint 參數來源

FastAPI 根據**型別**自動判斷參數來源：

```python
async def endpoint(
    user_id: str,           # Path 參數（在 URL 的 {user_id}）
    q: str | None = None,   # Query String（?q=xxx）
    body: UserUpdate,       # Request Body（Pydantic Model）
    redis = Depends(...),   # 依賴注入
    req: Request,           # 原始請求物件
    res: Response,          # 修改回應用（設 Cookie、Header）
):
```

### `req` 和 `res` 的用途

```python
# req：讀取請求資訊
req.headers.get("Authorization")
req.cookies.get("refresh_token")
await req.json()
req.client.host  # 客戶端 IP

# res：修改回應屬性（跟 return 分開）
res.set_cookie(key="token", value="...", httponly=True)
res.headers["X-Custom"] = "value"

# return：回應的 JSON body
return TokenResponse(access_token=new_token)
```

> `res` 和 `return` 最終由 FastAPI 合併成一個完整 HTTP Response

### 格式驗證

Pydantic Model 格式不符時自動回傳 **422 Unprocessable Entity**（不是 400）

## 10. 🧩 Python 概念補充

### `asynccontextmanager`

```python
from contextlib import asynccontextmanager

@asynccontextmanager
async def my_resource():
    # yield 之前：初始化
    resource = await setup()
    try:
        yield resource   # 暫停，把資源交出去
    finally:
        # yield 之後：清理（保證執行，即使發生錯誤）
        await cleanup()
```

### `yield` 兩種用法

```python
# 用法一：純暫停點（不交出值）
async def lifespan():
    await init()
    yield        # 暫停在這，等外部用完
    await cleanup()

# 用法二：交出值 + 暫停
async def get_redis():
    async with Redis(pool) as redis:
        yield redis  # 交出 redis 物件，同時暫停等 endpoint 用完
```

### `AsyncGenerator[YieldType, SendType]`

```python
async def get_redis() -> AsyncGenerator[Redis, None]:
#                                       ↑      ↑
#                                  yield的型別  send()型別（固定None）
    async with Redis(connection_pool=pool) as redis:
        yield redis
```

```
  │        ├─ redis.set() 存新 token

```
