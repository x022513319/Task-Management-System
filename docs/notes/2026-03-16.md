## 1. FastAPI vs Django 差異

Django 是 **batteries included**（電池內附），FastAPI 只給你快速的核心，其他全部自己選、自己接。

| 功能 | Django | FastAPI |
|------|--------|---------|
| 資料庫連線 | 自動 | 自己建 `engine` |
| Session 生命週期 | 自動 | 自己寫 `get_db()` |
| Model 母類別 | 內建 `models.Model` | 自己建 `Base` |
| 設定管理 | 內建 `settings.py` | 用 `pydantic-settings` |
| 認證 | 內建 `auth` app | 全部自己實作 |
| Migration | 內建 `makemigrations` | 裝 `alembic` |
| Admin 後台 | 內建 `/admin` | 無 |
| API 文件 | 需裝第三方套件 | 自動產生 `/docs` |

**FastAPI 優勢：** async 高併發效能、Pydantic 型別安全、自動 Swagger 文件、輕量彈性適合微服務。

## 2. 專案結構與職責

```
app/
├── api/            # Router，定義 endpoint
├── core/
│   ├── config.py   # 設定值，從 .env 讀取
│   └── security.py # JWT 驗證邏輯，get_current_user()
├── db/
│   ├── base.py     # 所有 model 繼承的母類別 Base
│   └── session.py  # engine、SessionFactory、get_db()
├── models/         # DB table 結構（SQLAlchemy）
├── schemas/        # API request / response 結構（Pydantic）
└── repositories/   # DB 查詢邏輯
```

### schemas vs models 核心概念

```
Request JSON → Pydantic Schema（驗證）→ SQLAlchemy Model（存 DB）
DB 查詢結果  → SQLAlchemy Model（取出）→ Pydantic Schema（序列化）→ Response JSON
```

**為什麼要分開？** 安全隔離。例如 `password` 明文進來（Schema），hash 後存 DB（Model），response 完全不輸出 `password`。

```python
# models/user.py — DB 層
class User(Base):
    hashed_password = Column(String)  # 只存 hash，不對外暴露

# schemas/user.py — API 層
class UserCreate(BaseModel):
    password: str           # 接收明文，但不存進 DB

class UserResponse(BaseModel):
    id: int
    email: str
    # password 完全不出現 ← 安全隔離
```


`BaseSettings` 會自動讀 `.env` 檔：

```python
from pydantic_settings import BaseSettings, SettingsConfigDict

class Settings(BaseSettings):
    model_config = SettingsConfigDict(env_file=".env")

    SQLALCHEMY_DATABASE_URL: PostgresDsn
    REDIS_DATABASE_URL: RedisDsn
    JWT_SECRET: str
    JWT_ALGORITHM: str = "HS256"
    ACCESS_TOKEN_EXPIRE_MINUTES: int = 30
    REFRESH_TOKEN_EXPIRE_DAYS: int = 7

settings = Settings()  # 在 config.py 建立，其他地方 import 使用
```

| 優點 | 說明 |
|------|------|
| 型別驗證 | 裸用 `os.getenv()` 全是字串，`BaseSettings` 自動轉型 |
| 預設值管理 | 集中一處，不散落各地 |
| 啟動時檢查 | 漏設環境變數，啟動時就報錯 |
| IDE 補全 | 有型別提示，`os.getenv()` 沒有 |

## 4. SQLAlchemy — 資料庫層

### 4-1 連線字串格式

格式：`dialect+driver://user:password@host:port/dbname`

| 連線字串 | 用途 |
|---------|------|
| `postgresql+asyncpg://...` | PostgreSQL，async（FastAPI 常用） |
| `postgresql+psycopg2://...` | PostgreSQL，同步（傳統 Flask/Django） |
| `sqlite+aiosqlite:///./app.db` | SQLite，async（本地開發測試） |

- **dialect** = 用哪種資料庫（`postgresql`, `mysql`, `sqlite`）
- **driver** = 用哪個 Python 套件連線（`asyncpg`, `psycopg2`, `aiosqlite`）

### 4-2 db/session.py

```python
from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession
from sqlalchemy.orm import sessionmaker
from app.core.config import settings

# engine 內部包含 connection pool，執行後還沒真正連線
engine = create_async_engine(str(settings.SQLALCHEMY_DATABASE_URL))

# sessionmaker 回傳的是 class（工廠），不是實例
# 每次 request 呼叫 AsyncSessionLocal() 才產生獨立 session
AsyncSessionLocal = sessionmaker(
    engine,
    class_=AsyncSession,
    expire_on_commit=False  # 見下方說明
)

async def get_db():
    async with AsyncSessionLocal() as session:
        yield session  # request 結束自動 close
```

> **`expire_on_commit=False`**：FastAPI 每個 request 結束會關 session。預設 `True` 的話，commit 後物件過期，再存取會因為找不到 session 而報錯。

> **`async def` vs `async with`**：`async def` 宣告這個 function 本身是 coroutine；`async with` 是因為 session 的開啟/關閉涉及網路 I/O，需要 await。

### 4-3 db/base.py

SQLAlchemy 版的 `django.db.models.Model`，所有 model 繼承它：

```python
from sqlalchemy.orm import DeclarativeBase

class Base(DeclarativeBase):
    pass  # 可在此加共用欄位，如 created_at、updated_at
```

### 4-4 Model 定義

```python
from sqlalchemy.dialects.postgresql import UUID
from sqlalchemy import String, func
from sqlalchemy.orm import Mapped, mapped_column, relationship
import uuid
from datetime import datetime

class User(Base):
    __tablename__ = "users"  # 慣例用複數

    id: Mapped[uuid.UUID] = mapped_column(
        UUID(as_uuid=True),
        primary_key=True,
        default=uuid.uuid4  # 不需要 lambda
    )
    username: Mapped[str] = mapped_column(String(64))
    password_hash: Mapped[str] = mapped_column(String(128))

    # Mapped[str] 已代表 NOT NULL，不需再寫 nullable=False
    # Mapped[Optional[str]] 自動對應 nullable=True

    created_at: Mapped[datetime] = mapped_column(server_default=func.now())
    updated_at: Mapped[datetime] = mapped_column(
        server_default=func.now(),
        onupdate=func.now()
    )

    tasks: Mapped[List["Task"]] = relationship(back_populates="user")
```

> **UUID 用 PostgreSQL 原生型別**：`UUID(as_uuid=True)` 佔 16 bytes，比 `String(36)` 模擬（36 bytes）效能更好，且 DB 層有型別驗證。

> **`server_default` vs `default`**：`server_default` 讓 DB 產生時間，直接執行原生 SQL 繞過 SQLAlchemy 時也能觸發，比 `default` 更可靠。

### 4-5 關聯（ForeignKey + relationship）

| | 層面 | 作用 |
|--|------|------|
| `ForeignKey` | DB 層 | 告訴資料庫「這欄位參照另一張 table 的 id」 |
| `relationship` | Python 層 | 讓你直接用物件存取關聯資料，SQLAlchemy 自動產生 JOIN |

```python
class Task(Base):
    __tablename__ = "tasks"

    id: Mapped[uuid.UUID] = mapped_column(UUID(as_uuid=True), primary_key=True, default=uuid.uuid4)
    title: Mapped[str] = mapped_column(String(128))

    user_id: Mapped[uuid.UUID] = mapped_column(ForeignKey("users.id"))  # DB 欄位
    user: Mapped["User"] = relationship(back_populates="tasks")          # Python 存取
```

`back_populates` 讓兩邊互相連結：

```python
task.user_id   # → UUID（DB 欄位）
task.user      # → User 物件（relationship，自動執行 SELECT）
user.tasks     # → List[Task]（relationship）
```

> **1 對多**：FK 要放在「多」的那邊（Task），不是「一」的那邊（User）。

### 4-6 DB 操作流程

```python
user = User(name="Alice")   # 建立 Python 物件，尚未操作 DB
session.add(user)           # 加入 session 待處理清單
await session.commit()      # 真正執行 INSERT INTO users...
```

為什麼分三步？可以一次 commit 多個操作，保證原子性（要嘛全成功，要嘛全失敗）。

## 5. Docker Compose

### 5-1 基本結構

```yaml
# 現代 Docker Compose V2 不需要寫 version
services:
  db:
    image: postgres:16        # 指定版本，不用 latest
    shm_size: 128mb           # 共享記憶體，PostgreSQL 多 process 溝通用
    environment:
      POSTGRES_USER: ${POSTGRES_USER}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
      POSTGRES_DB: ${POSTGRES_DB}
    ports:
      - "5432:5432"
    volumes:
      - db_data:/var/lib/postgresql/data  # 注意是 postgresql 不是 postgres

volumes:
  db_data:  # 必須在這裡宣告
```

環境變數從 `.env` 讀取，Docker Compose 自動讀同目錄的 `.env`。  
記得把 `.env` 加入 `.gitignore`，提供 `.env.example` 給其他人參考。

### 5-2 常用指令

| 指令 | 說明 |
|------|------|
| `docker compose up -d` | 背景執行（detached mode） |
| `docker compose up -d --build` | 強制重新 build image（改了 Dockerfile 或新增套件時用） |

### 5-3 Volumes 兩種類型

| 類型 | 範例 | 說明 |
|------|------|------|
| named volume | `db_data:/var/lib/postgresql/data` | Docker 自己管理，container 刪掉資料還在 |
| bind mount | `./data:/var/lib/postgresql/data` | 直接對應本機資料夾 |

> **`shm_size`**：`/dev/shm` 是 Linux 共享記憶體，PostgreSQL 的多個 process 用來快速交換資料。Docker 預設只有 64MB，開發環境設 `128mb` 即可。

## 6. 密碼處理（bcrypt）

`bcrypt.hashpw()` 需要 `bytes`，不能直接傳 `str`：

```python
from bcrypt import hashpw, gensalt, checkpw

def set_password(self, password: str) -> None:
    self.password_hash = hashpw(
        password.encode("utf-8"),
        gensalt()
    ).decode("utf-8")

def verify_password(self, password: str) -> bool:
    return checkpw(
        password.encode("utf-8"),
        self.password_hash.encode("utf-8")
        # checkpw 是 bcrypt 內建驗證，比手動比較 hash 更正確
    )
```
